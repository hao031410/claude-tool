## 目录结构与层次职责

### 六边形架构

```
模块结构：
├── spi          # 元数据、PO、DTO、Converter、Dict
├── adapter      # Action（REST Controller）
├── app          # AppService、业务编排、事务
├── domain       # DomainService、领域逻辑
├── infrastructure # Repository、Gateway（外部调用）
└── starter      # 启动类、配置
```

### 层次依赖规则

**严格单向依赖**：
```
starter → adapter → app → domain → infrastructure → spi
```

**禁止反向依赖**：
- ❌ infrastructure 不能依赖 app
- ❌ domain 不能依赖 infrastructure
- ❌ 各层不能跨层依赖

## 模型定义规范

### Java类位置

```
io.terminus.erp.{module}.spi.model.po.{Entity}PO
io.terminus.erp.{module}.spi.model.dto.{Entity}DTO
```

### PO (Persistent Object)

**继承 BaseModel**：
```java
@Data
@EqualsAndHashCode(callSuper = true)
@TableName("sett_item_tr")
public class SettItemTrPO extends BaseModel {
    private static final long serialVersionUID = -65282004572364214L;

    @ApiModelProperty("结算项编码")
    @TableField("`sett_item_code`")
    private String settItemCode;
}
```

**模型关联标识**：
```java
// 在模型中标识关联
SettItemTrPO.inv_org.id // 关联 org_struct_md
```

### JSON字段定义

**标准格式**：
```json
[
  {
    "key": "sett_item_code",
    "name": "结算项编码",
    "props": {
      "fieldType": "TEXT",
      "required": false,
      "unique": false,
      "compositeKey": false,
      "isSystemField": false,
      "autoGenerated": false,
      "comment": "结算项编码",
      "columnName": "sett_item_code",
      "length": 32,
      "encrypted": false,
      "relationField": false,
      "i18nEnabled": false
    }
  }
]
```

### Converter模型转换

**MapStruct接口**：
```java
@Mapper(componentModel = "spring")
public interface SettItemTrConverter {
    SettItemTrDTO po2dto(SettItemTrPO po);
    SettItemTrPO dto2po(SettItemTrDTO dto);
    List<SettItemTrDTO> po2dtoList(List<SettItemTrPO> poList);
    List<SettItemTrPO> dto2poList(List<SettItemTrDTO> dtoList);
}
```

### Dict枚举值

**强制要求**：PO/DTO中如果使用枚举，必须注释并用 `@see` 标识具体枚举值。

```java
/**
 * 结算项状态
 * @see SettItemStatusDict
 */
@TableField("sett_item_status")
private String settItemStatus;
```

## 各层职责

### Repository（infrastructure层）

**职责**：数据访问、基本CRUD

**命名**：`{Entity}Repo`

**继承**：`extends BaseRepository<PO>`

**查询规范**：
```java
// ✅ 推荐：使用 LambdaQueryWrapper
default List<SettItemTrPO> findByStatus(String status) {
    return selectList(new LambdaQueryWrapper<SettItemTrPO>()
        .eq(SettItemTrPO::getStatus, status));
}

// ❌ 禁止：硬编码字段名
selectList(new QueryWrapper<SettItemTrPO>()
    .eq("status", status))
```

### DomainService（domain层）

**职责**：领域复杂DAO封装（统一事务提交、多模型联合查询）

**返回**：一般用DTO返回

**命名**：`{Entity}DomainService`

### AppService（app层）

**职责**：复杂业务逻辑编排、事务管理

**规范**：
- 抽象的CRUD建议放到domain中
- App多做业务相关处理
- 简单查询允许直接调用Repo
- **增删改必须走domain**

**事务注解**：
```java
@Transactional(rollbackFor = Exception.class)
public class SettDocTrSubmitAppService {
    // ...
}
```

### Action（adapter层）

**职责**：Web入口、REST接口

**注解**：`@Action`（Trantor2）

**返回**：`Response<T>`格式

```java
@Action
public class SettDocSubmitAction {
    @PostMapping("/submit")
    public Response<Void> submit(@RequestBody SettDocTrDTO dto) {
        // ...
        return Response.ok();
    }
}
```

### Gateway（infrastructure层）

**职责**：查询其他模块

**规范**：
- 必须先引入依赖（Maven）
- 按需定义方法，byId等方法尽可能复用

**示例**：
```java
@Component
public class SettMdGateway {
    @TService("SETT_MD_SERVICE")
    private MdServiceClient mdServiceClient;

    public GenMatMdPO getMatById(Long id) {
        return mdServiceClient.getById(id);
    }
}
```

**依赖引入**：
```xml
<dependency>
    <groupId>io.terminus.erp</groupId>
    <artifactId>erp-onlinemall-infrastructure</artifactId>
    <version>${project.version}</version>
</dependency>
```

## 消息队列与定时任务

### 消息队列

**消费**：
```java
@Slf4j
@MQConsumer(consumerGroup = "group")
public class MqConsumer {
    @MQSubscribe(topic = "topic", tag = "tag")
    public void consume(Message message) {
        // 处理消息
    }
}
```

**发送**：
```java
@Autowired
private TerminusMQProducer producer;

public void sendMessage() {
    TerminusMessage message = new TerminusMessage();
    message.setBody(new Object());
    message.setTags("tag");
    message.setTopic("topic");
    producer.send(message);
}
```

### 定时任务

**代码**：
```java
@Slf4j
@Component
public class JobHandler {
    @Job("job_demo")
    public void execute(Object data) {
        // 任务逻辑
    }
}
```

**job_definition.json**（通常在app目录）：

**1. SecondDelay（延时任务）**：
```json
{
  "module": "ERP_SETT",
  "jobKey": "FIN_ASYNC_EXECUTION_JOB",
  "jobName": "财务异步任务",
  "jobType": "SecondDelay",
  "execType": "Default",
  "status": "ENABLED"
}
```

**2. Crontab（定时任务）**：
```json
{
  "module": "ERP_FIN",
  "jobKey": "BI_HISTORY_CLEAN_JOB",
  "jobName": "报表历史数据清除任务",
  "jobType": "Crontab",
  "execType": "Default",
  "crontab": "1 0 0 ? * MON",
  "params": "{}",
  "extraInfo": "{}",
  "status": "ENABLED"
}
```

**字段说明**：
- `jobType`: `SecondDelay`（延时）或 `Crontab`（定时）
- `crontab`: 仅 Crontab 类型需要，遵循 Cron 表达式格式（秒 分 时 日 月 周）
- `params`: 任务参数（JSON 格式）
- `extraInfo`: 扩展信息（JSON 格式）
- `status`: `ENABLED`（启用）或 `DISABLED`（禁用）

**Cron 表达式示例**：
- `1 0 0 ? * MON` - 每周一 00:00:01 执行
- `0 0 12 * * ?` - 每天 12:00:00 执行
- `0 0/30 * * * ?` - 每 30 分钟执行一次
